# fee

검침 청구 단체문자 웹 MVP입니다.

한셀/Excel/CSV 시트를 불러와 수신자별 청구 상세 카드를 만들고, 문자 앱으로 발송 큐를 여는 정적 웹앱입니다.

## 사용 방법

1. `index.html`을 브라우저에서 엽니다.
2. `.csv`, `.xls`, `.xlsx` 파일을 선택하거나 끌어놓습니다.
3. 단체문자 문구를 작성합니다.
4. `{조회링크}`(또는 `{상세링크}`) 변수를 문자에 포함합니다.
5. 관리자 비밀번호를 입력하고 `본인조회 데이터 게시`를 눌러 조회 데이터를 클라우드에 올립니다. (아래 "본인 조회 데이터 공유" 참고)
6. `발송 큐 만들기`를 눌러 수신자별로 문자 앱을 여는 순서를 확인합니다. 큐는 항상 1명씩 개별 SMS로 열립니다.

## 시트 헤더

필수 발송용 헤더:

- 전화번호

전화번호는 Excel/한셀에서 앞자리 0이 사라지지 않도록 텍스트 형식으로 저장하는 것이 가장 안전합니다. 앱은 `1012345678`처럼 읽힌 국내 휴대폰 번호를 `01012345678`로 자동 보정합니다.

카드 표시용 헤더:

- 소속
- 성명
- 거주지명
- 호수
- 전월검침
- 당월검침
- 당월사용량
- 국고지원량
- 초과사용량
- 청구금액
- 국고지원금
- 초과금액
- 사용인원
- 인원별부담금액

호수·검침값·금액·인원 등 숫자 항목은 가져오기/직접 입력 시 "호", "kWh", 쉼표 등 문자를 자동으로 제거하고 숫자만 남깁니다. (예: `101호` → `101`) CSV 파일 맨 앞의 BOM 문자도 자동으로 제거되어 한셀에서 내보낸 CSV의 첫 헤더가 깨지는 문제를 방지합니다.

## 본인 조회 데이터 공유

이 저장소는 public GitHub 저장소이므로, 조회용 개인정보(성명/전화번호 뒷자리/금액 등)를 저장소 파일이나 GitHub Pages에 그대로 올리면 검색으로 노출될 수 있습니다. 대신 Supabase(무료 티어) DB에 저장하고, 수신자는 이름+호수+전화번호 뒷4자리가 모두 일치할 때만 RPC 함수를 통해 본인 데이터 1건만 조회할 수 있도록 구성했습니다.

- 테이블(`billing_recipients`, `billing_admin_secret`)은 RLS로 직접 조회/수정이 막혀 있고, `billing_lookup_recipient`(조회)와 `billing_replace_recipients`(게시)라는 두 RPC로만 접근합니다.
- 관리자 비밀번호는 최초 게시 시 서버에 등록되고, 이후 게시할 때는 같은 비밀번호를 입력해야 합니다(비밀번호가 다르면 게시가 거부됩니다).
- 게시할 때마다 기존 조회 데이터는 전체 교체됩니다(최신 시트만 유지).

## SMS vs MMS

휴대폰은 문자 앱이 아래 두 조건 중 하나라도 해당하면 자동으로 그룹/장문 MMS로 전환하며, 이 경우 통신사에 따라 요금이 발생할 수 있습니다.

1. 수신자가 2명 이상 묶인 문자 — 그래서 발송 큐는 항상 수신자 1명당 1건으로 열립니다(묶음 발송 기능 제거).
2. 문구가 약 80바이트(한글 약 40자)를 넘는 장문 — 문구 작성 칸 아래 바이트 안내를 참고해 짧게 유지하세요.

웹 브라우저는 보안상 SMS를 자동 발송할 수 없습니다. 이 앱은 문자 내용을 만들고 휴대폰 문자 앱을 여는 방식으로 동작합니다.
